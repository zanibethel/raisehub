# 1. Communication Style

## Beginner-Friendly Guidance

Zac prefers:

- Plain-language explanations
- Clear file paths
- Complete commands
- Step-by-step instructions
- Minimal unexplained jargon
- Direct explanations of what to expect after each action

When a technical term is necessary, explain it briefly.

Do not assume familiarity with:

- Terminal navigation
- Git internals
- Supabase administration
- Next.js conventions
- TypeScript type relationships
- Deployment infrastructure
